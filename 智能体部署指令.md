## ⚠️ 重要：必须执行的部署步骤

本技能需要**两步部署**才能完整生效：

1. **安装技能包** - 复制调研和 PPT 生成脚本
2. **更新 SOUL.md** - 添加新客户拜访铁律

**缺一不可。**

---

## 步骤 1：安装技能包

### 方法 A：通过 ClawHub 安装（推荐）
```bash
openclaw skills install customer-research
```

### 方法 B：手动复制
```bash
cp -r /root/.openclaw/workspace/skills/customer-research ~/.openclaw/workspace/skills/
```

### 验证安装
```bash
ls ~/.openclaw/workspace/skills/customer-research/
# 应看到：SKILL.md, scripts/, references/
```

### 依赖检查
确保以下技能已安装：
```bash
openclaw skills install word-docx
openclaw skills install openclaw-ppt-generator
```

---

## 步骤 2：更新 SOUL.md

**必须**在你的 `SOUL.md` 中添加以下章节：

```markdown
---

## 🎯 新客户拜访铁律

**首次拜访前必须执行客户调研，无例外。**

### 调研输出（强制交付）
| 交付物 | 内容 | 用途 |
|--------|------|------|
| **Word 调研报告** | 客户背景、AI 领导发言、招标记录、数据相关 | 销售内部准备 |
| **场景破冰 PPT** | before/after 具体场景对比（6-8 页） | 拜访时演示 |

### PPT 标准
- ✅ 4 个以上具体业务场景（如"院长经营看板""护士排班"）
- ✅ 每个场景 before/after 对比（细节描述，不是空话）
- ✅ 量化指标（时间/金额/百分比，如"3 天→实时"）
- ❌ 禁止"数字化转型""赋能""提质增效"等空泛词汇

### 执行时机
- 销售说"要去拜访 XX 客户"→ 立即触发调研
- 销售说"准备见 XX 客户"→ 立即触发调研
- 销售说"调研 XX 客户"→ 立即触发调研
- 新客户首次出现→ 主动询问是否需要调研

### 调研技能位置
`~/.openclaw/workspace/skills/customer-research/`
```

### 自动更新命令（推荐）
```bash
# 检查 SOUL.md 是否已包含新客户拜访铁律
if ! grep -q "新客户拜访铁律" ~/.openclaw/workspace/SOUL.md; then
    # 追加到 SOUL.md
    cat /root/.openclaw/workspace/skills/customer-research/SOUL_APPEND.md >> ~/.openclaw/workspace/SOUL.md
    echo "✅ SOUL.md 已更新"
else
    echo "⚠️ SOUL.md 已包含新客户拜访铁律"
fi
```

## 🎯 核心执行规则

### 触发词（识别到立即执行）
- "要去拜访 XX 客户"
- "准备见 XX 客户"
- "调研 XX 客户"
- "帮我查一下 XX 客户"
- "XX 客户的情况"

### 禁止行为（违反=销售事故）
1. ❌ 禁止问"要不要调研" - 直接执行
2. ❌ 禁止务虚空话 - "数字化转型""赋能""提质增效"
3. ❌ 禁止只给文件路径 - 必须用 `openclaw message send --media` 发送
4. ❌ 禁止编造信息 - 无来源时明确告知"公开渠道未检索到"

### 输出标准
| 文件 | 命名格式 | 发送方式 |
|------|----------|----------|
| 调研报告 | `[客户名称]_调研报告.docx` | `openclaw message send --media` |
| 场景破冰 PPT | `[客户名称]_场景破冰 PPT.pptx` | `openclaw message send --media` |

## 📊 PPT 场景标准

### Before 写法（痛点细节）
**❌ 错误：**
- "数据不准确"
- "效率低"

**✅ 正确：**
- "每月 5 号等财务报表，各院区 Excel 汇总要 3 天"
- "每周花 4 小时排班，要平衡 20+ 护士的班次"

### After 写法（价值细节）
**❌ 错误：**
- "提升效率"
- "智能排班"

**✅ 正确：**
- "每天早上 8 点自动推送昨日经营日报"
- "自动排班 + 一键调整，30 分钟完成"

### 量化指标
- 时间类：3 天 → 实时、4 小时 → 30 分钟
- 百分比类：70% → 85%、65% → 85%
- 金额类：10 万/年 → 1 万/年

## 🔧 执行流程

```
1. 识别触发词 → 提取客户名称
2. 执行 4 轮搜索：
   - AI 相关领导发言
   - AI 相关招标记录
   - 数据相关内容
   - 基本情况与最新动态
3. 生成 Word 报告（create_word.py）
4. 生成场景破冰 PPT（create_scenario_ppt.py）
5. 发送文件（openclaw message send --media）
6. 记录客户记忆（memory/customers/[客户名称].md）
7. 询问销售下一步行动
```
