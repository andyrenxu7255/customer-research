## 各位销售智能体同事：

这是统一开发的**客户调研技能**，用于规范所有销售智能体的客户拜访前准备流程。

**核心目标：** 让每个销售在拜访客户前，都有充分的调研材料和场景化破冰 PPT。

## 🎯 核心规则（必须遵守）

### 铁律
**新客户拜访前必须执行调研，无例外。**

### 触发词
销售说以下任意一句，立即触发调研：
- "要去拜访 XX 客户"
- "准备见 XX 客户"
- "调研 XX 客户"
- "帮我查一下 XX 客户"
- "XX 客户的情况"

**不要问"要不要调研"，直接执行。**

### 输出文件
| 文件 | 命名格式 | 用途 |
|------|----------|------|
| 调研报告 | `[客户名称]_调研报告.docx` | 销售内部准备 |
| 场景破冰 PPT | `[客户名称]_场景破冰 PPT.pptx` | 拜访时演示 |

## 📊 PPT 标准（严格执行）

### 页面结构（6-8 页）
| 页码 | 主题 |
|------|------|
| P1 | 封面：[客户名称] 业务场景洞察 |
| P2 | 场景 1：before/after 对比 |
| P3 | 场景 2：before/after 对比 |
| P4 | 场景 3：before/after 对比 |
| P5 | 场景 4：before/after 对比 |
| P6 | 验证计划：5 周 POC 路径 |

### Before 写法（痛点细节）
**❌ 错误示例：**
- "数据不准确"
- "效率低"
- "客户流失"

**✅ 正确示例：**
- "每月 5 号等财务报表，各院区 Excel 汇总要 3 天"
- "每周花 4 小时排班，要平衡 20+ 护士的班次"
- "产后 42 天复查靠护士打电话，100 个患者打半天"

### After 写法（价值细节）
**❌ 错误示例：**
- "提升效率"
- "智能排班"
- "提高随访率"

**✅ 正确示例：**
- "每天早上 8 点自动推送昨日经营日报"
- "自动排班 + 一键调整，30 分钟完成"
- "系统自动推送复查提醒，随访率提升到 85%+"

### 量化指标要求
- 时间类：3 天 → 实时、4 小时 → 30 分钟
- 百分比类：70% → 85%、65% → 85%
- 金额类：10 万/年 → 1 万/年

## 🔧 使用示例

### 输入
```
销售：要去拜访美中宜和
```

### 智能体执行
```
1. 识别客户名称：美中宜和
2. 执行 4 轮搜索（AI 领导发言、招标记录、数据相关、基本情况）
3. 生成 Word 调研报告
4. 生成场景破冰 PPT（before/after 对比）
5. 发送文件到企业微信
6. 记录到 memory/customers/美中宜和.md
7. 询问销售下一步行动
```

### 输出
```
📊 美中宜和调研报告已生成，请查收。
📊 美中宜和场景破冰 PPT 已生成，请查收。

销售，调研完成。下一步：
1. 预约拜访信息科负责人
2. 准备拜访问题清单
3. 还是有其他安排？
```

## ⚠️ 禁止行为（违反=销售事故）
